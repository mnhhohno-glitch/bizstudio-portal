# 求人マッチングアドバイザースキル

## 概要

15年のキャリアアドバイザー経験に基づく求人マッチングの判断フレームワークと、業界標準のベストプラクティスを融合したスキル。
主な対象層は20代前半〜30代前半の若手(特に第二新卒領域)、オフィスワーク系職種(営業・事務・技術・管理部門)、年収帯300万〜600万円。転職回数は初回60%・2回目30%・3回目以上10%。

このスキルは以下の7つのフェーズで構成される。

### 最重要原則:面談ログに書かれていないことは存在しない

面談ログや経歴書を分析する際、以下を厳守すること。このルールは他の全てのルールに優先する。

**大原則:** 面談ログに明示的に記載されていない情報を分析結果に含めてはいけない。「一般的にこう考えるはず」「このタイプの人はこういう条件を重視するはず」「結婚したから安定志向のはず」といった推測で条件や希望を追加することは禁止。

- **条件の捏造禁止**:面談で話されていない希望条件・NG条件・年収下限・リモート希望・残業条件を勝手に追加しない。面談ログで確認されていない条件は「次回確認が必要」と書く
- **発言の捏造禁止**:ログに存在しない発言を「〜と回答した」と記載しない。根拠は必ずログの原文から
- **発言の意味を反転させない**:「土日休みなら残る」を「残らない」と解釈するような誤読をしない
- **一般論と具体的指示を混同しない**:CAが「下がる可能性がある」と説明したことを「450万を指示した」としない
- **提案のトーン・強度を正確に読む**:「次回メインで提案」と「2〜3件混ぜる」と「保留」は全く違う温度感
- **属性ベースの推測でタイプ判定しない**:「結婚したから安定志向」「女性だから安定志向」等、面談での発言ではなく属性から推測してタイプを判定してはいけない。「キャリアアップしたい」と言っている人を「安定志向」と判定しない
- **推測が含まれる場合は明記する**:「面談ログからは読み取れないが〜の可能性がある(※推測)」と区別する

## Phase 1: 書類の読み方 — 情報源ではなく行動の現れとして読む

### 基本原則

職務経歴書やレジュメを「スキルシート」としてではなく、**その人の人生の意思決定の記録**として読む。書類から読み取るべきは「何ができるか」だけではなく「どんな判断をしてきた人か」。

### 初見のアプローチ(3ステップ俯瞰法)

1. **全体俯瞰**:転職回数と在籍期間のパターンからキャリアの一貫性・意思決定の傾向を見る
2. **意思決定の軸を読む**:なぜこの会社に入り、なぜ辞めたのかの繰り返しから思考性・志向性を掴む
3. **直近の職歴を確認**:現在のスキルレベルとポジションを把握する

### 書類から読み取る「温度感と性格」

正確で完成度の高い職務経歴書が初回面談前に出てくることは少ない。そのため、書類からは内容の正確性よりも**取り組み姿勢**を読む。

| 観察ポイント | 読み取れること |
|---|---|
| **記入量** | 多い=転職への本気度が高い、少ない=温度感が低いか大雑把な性格 |
| **書き方の丁寧さ** | 細かく整理=几帳面、雑=大雑把か転職活動初期段階 |
| **自己PR** | 内容は読まない。文字量だけでパッと見判断(熱量の指標) |
| **実績記載** | 誇張が多いため額面通りには受け取らない |

### 温度感判定の分岐フロー(重要)

書類の記入量が少ない場合、「やる気がない=温度感が低い」と即断してはいけない。**まず「書けないから少ないのか、書かないから少ないのか」を判定する。**

**判定の分岐条件:**

1. **経歴を確認**:民間企業でのビジネス経験があるか
   - 民間企業経験あり → 書類作成の経験があるはず → 記入量が少ない=温度感が低い可能性
   - 民間企業経験なし → 下記へ

2. **非ビジネス経歴かどうか**:以下に該当する場合は「書き方がわからないだけ」の可能性が高い
   - 自衛隊・警察・消防出身
   - 教師・公務員出身
   - 高卒で特殊業界のみの経験
   - アルバイト経験のみ

3. **面談での態度と照合**:書類は簡素でも、面談で積極的に質問する・日程をすぐ決める・他社にも応募済みなど、行動面で温度感が高ければ「書けないだけ」と確定する

**原則:学歴・経歴・年齢から「ビジネス文書の経験があるか」を先に判断し、経験がない場合は記入量の少なさを温度感と結びつけない。**

### 温度感と「慎重タイプ」の見分け

温度感が低く見えても、実際は**慎重さが行動を止めているだけ**のケースがある。特に以下に該当する場合は「慎重タイプ」を疑う:

- **過去に転職活動で失敗(不合格)した経験がある**:挫折体験がトラウマとなり動けない
- 「良いところがあればすぐにでも」と言いつつ「転職したくないけどせざるを得ない」というニュアンス
- 紹介経由で来ている(自分からエージェントに登録する踏ん切りがつかなかったが、背中を押されて動いた)

慎重タイプは温度感が低いのではなく、**失敗への恐怖が行動を止めている**。対応としては、通過率の高い求人で早期に成功体験を積ませることが有効。

### 紹介経由・書類なしケースの分析手法

求職者からの紹介で面談に来た場合、レジュメやWeb履歴書がないケースが多い。その場合は**面談ログのみで以下を抽出する**:

1. **基本スペック**:年齢、最終学歴、現職の社名・業種・職種、勤続年数、転職回数
2. **業務内容の具体性**:面談で自分の仕事をどれだけ具体的に説明できるかで、業務理解度と言語化力を測る
3. **退職理由の構造**:一時的な不満か構造的な問題か
4. **希望条件**:年収・勤務地・休日・残業・転勤の可否
5. **性格・温度感**:話し方、敬語の使い方、日程調整の即応性、質問の積極性から読む

**注意:書類がなくても面談ログから十分なタイプ診断・検索戦略の策定は可能。ただし業務内容の詳細は面談後にテキストで箇条書きしてもらい、経歴書作成に繋げる。**

### 【一般的CA知見】ポータブルスキルの抽出

書類からは、業界・職種固有のスキルだけでなく、**ポータブルスキル**(業界を跨いでも活かせる汎用スキル)を抽出することが重要。特に第二新卒領域では職種経験が浅いため、以下の観点で読む:

- **対人スキル**:顧客折衝、社内調整、プレゼン経験
- **課題設定・解決力**:業務改善、プロジェクト推進の経験
- **数値管理力**:売上管理、KPI追跡、データ分析
- **セルフマネジメント**:自律的に動いてきた経験の有無

これらは求人票の必須要件には明記されないが、実際の選考では重視されることが多い。

## Phase 2: タイプ診断 — Will-Can-Mustフレームワーク × 6タイプ志向性分析

### Will-Can-Must による構造分析

求職者を理解するための基本フレームワークとして、業界標準のWill-Can-Mustを使う。

- **Will(やりたいこと)**:本人が希望する方向性、キャリアビジョン
- **Can(できること)**:現在保有するスキル・経験・実績
- **Must(求められること)**:市場が求める人材要件、現実的な選択肢

**重要:3つが重なる領域が最も現実的なマッチングゾーン。** ただし、Willが先行しすぎてCanが伴わないケース(未経験職種への希望)や、CanはあるがWillがない(今の仕事はできるが嫌だ)ケースを正確に見極める必要がある。

### 6タイプ志向性分析

Will-Can-Mustの構造分析に加え、求職者が仕事に魅力を感じるドライバーを以下の6タイプで分類する。

| タイプ | 特徴 | 求人マッチングへの影響 |
|---|---|---|
| **成果・達成型** | 数字を追う、目標達成に燃える | 明確なKPI・インセンティブがある求人 |
| **専門深化型** | 一つの領域を極めたい、職人気質 | 専門性を深められるポジション、資格取得支援 |
| **影響・裁量型** | 自分の判断で動ける範囲が広いほど活き活き | 裁量の大きいポジション、少数精鋭組織 |
| **関係構築型** | 人との関わり、チームワークが原動力 | チーム営業、カスタマーサクセス、社内調整系 |
| **安定・環境型** | 働き方や条件面の安心感が土台 | 大手・老舗企業、福利厚生充実、残業少 |
| **カルチャー重視型** | 企業の雰囲気やブランドに惹かれる | ネームバリューのある企業、社風が明確な企業 |

### 安定・環境型とカルチャー重視型の見分け方

一見似ているが提案する求人が変わる。同じ「大手志向」でも本当に安定を求めているのか、「知名度のある会社にいる自分」というブランドに惹かれているのかで異なる。後者ならベンチャーでも知名度が高ければマッチする可能性がある。

### タイプ判定のタイミング

- **書類段階**:転職回数・在籍期間・記入姿勢から仮説を立てる
- **面談中**:逆転質問などで検証・確定する
- **面談後**:言語化された希望条件と、経歴から読み取れる本音のズレを分析する

### 【一般的CA知見】複合タイプへの対応

多くの求職者は単一タイプではなく、複数の志向性を持つ。たとえば「成果・達成型 × 影響・裁量型」の人は、数字を追いながら自分の判断で動ける環境を求める。**主タイプと副タイプの組み合わせ**で求人のフィット度が変わるため、1タイプに決めつけず2〜3のタイプの優先順位で整理するのが望ましい。

## Phase 3: 面談テクニック — 本音を引き出す対話設計

### 面談の全体構成(一般的CA知見 + 独自手法)

| ステップ | 目的 | 手法 |
|---|---|---|
| 1. アイスブレイク | 緊張をほぐし、信頼の基盤を作る | 共通点の発見、転職活動の状況確認 |
| 2. キャリアの棚卸し | 経歴の事実確認と意思決定パターン把握 | 時系列で入社理由・退職理由を聞く |
| 3. 逆転質問 | 本当の転職理由と譲れない条件の特定 | 下記の「コア質問」参照 |
| 4. Will-Can-Mustの整理 | キャリアの方向性を明確化 | 本人の言語化を支援しつつ、CAが補完 |
| 5. 市場感のすり合わせ | 希望と現実のギャップを共有 | 相場感・通過率データの提示 |
| 6. 検索条件の合意 | 求人検索の方向性を握る | 優先順位の明確化 |

### コア質問:「どうだったら辞めなかったか」

転職理由を聞いた後に、以下の逆転質問を行う。

- 「今の会社がどういう状況だったら辞めなかったですか?」
- 「今の会社に残っていたら、今現在どういう未来になっていたと思いますか?」

#### この質問で得られる情報

1. **本当に譲れない条件**:辞めない条件=その人にとっての最優先事項
2. **転職の本気度と根深さ**:
   - 「上司が変われば辞めなかった」→ 一時的なトリガー、現職で解決可能かもしれない
   - 条件が複数積み重なる場合 → 構造的な不満、現職では解決不可能

### 年収に対する本気度の検証

求職者は「年収は下がってもいい」と最初に言いがちだが、**年収は最後に覆る条件の第一位**。

具体的な金額を突きつけて検証する:
- 「仮に年収がXX万円だったらどうしますか?」
- 即答で「大丈夫です」→ 信頼できる
- 「ちょっと考えます」「実は他社も受けています」→ 警戒すべき

### 【一般的CA知見】「求職者の魅力」の引き出し

面談は求人マッチングのための情報収集だけでなく、**求職者自身が気づいていない強みを発見する場**でもある。特に第二新卒では自己評価が低い傾向があるため、以下の観点で魅力を引き出す:

- 「その業務で一番工夫したことは何ですか?」→ 仕事への取り組み姿勢
- 「周囲から頼られることは何ですか?」→ 他者から見た強み(ジョハリの窓の「盲点の窓」)
- 「嬉しかったお客様の反応は?」→ 仕事のやりがいの源泉

ここで引き出した情報は、推薦状作成や面接対策にも活用できる。

### 危険シグナルの検知

| シグナル | 読み取り | 対応 |
|---|---|---|
| **面接のリスケ** | 他社選考の進展や志望度低下の可能性が高い | 2の手、3の手の準備 |
| **「考えます」の多用** | 他の選択肢を温めている | 具体的な懸念を引き出す |
| **条件をどんどん追加する** | 転職の軸が定まっていない、または決断を先延ばし | Will-Can-Mustに立ち返って整理 |
| **全ての求人に「いいですね」** | 判断基準が弱い、または本音を言えていない | Cランクの求人で反応を試す |
| **求人紹介後に連絡頻度が落ちる** | 転職活動の温度感が下がっている、または他社決定済みの可能性 | 早期にフォロー、状況確認 |
| **「やはり転職をやめます」と言ってくる** | 紹介自体は適切でも、求職者側の事情変化で動けなくなった | 引き留めは無理せず、関係維持を優先 |

### 紹介後の求職者状態の把握

求人紹介してから本人がエントリー意思を示すまでには数日〜数週間のタイムラグが発生する。この期間中の求職者の動きは以下の3パターンに分かれる:

- **動いている**:マイページを見て検討中、近日中にエントリー判断する見込み
- **動いていない(意思変化)**:転職活動自体を中止、他社決定、連絡途絶
- **動いていない(検討長期化)**:検討に時間がかかっている、判断材料が足りない

**紹介後3日経ってもマイページ閲覧の兆候がない場合は要フォロー**。連絡を入れることで、求職者側の状態が明確になり、CAとして次の手を打てる。

### 20代世代の「隠れ承認欲求」パターン

現在の20代に多いタイプとして、**承認欲求は非常に高いが表に出さない**パターンがある。特徴:

- 表面上は控えめで丁寧、自己主張が少ない
- しかし内面では「認められたい」「正当に評価されたい」という欲求が強い
- その分**打たれ弱く、面接不合格のダメージが大きい**
- 一度挫折すると1年以上動けなくなることもある

**対応:**
- 面接不合格時のフォローを手厚くする(「あなたの問題ではなくタイミングの問題」と伝える)
- 書類通過や面接通過のたびにしっかり承認する
- 市場価値を客観的データで示し、自信の根拠を与える

## Phase 4: 求人検索戦略

### 生活面の絶対制約条件(最優先・検索前に必ずチェック)

検索を始める前に、求職者に以下の生活面の時間的制約がないかを確認する。これがある場合、タイプ診断・年収条件・業種希望よりも優先度が高い絶対条件となる。この条件を満たさない求人はどんなにマッチしていても提案不可。

- **保育園・幼稚園・学童のお迎え時間**:例)18時までにお迎え → 退社17時必須。勤務地からお迎え先までの移動時間を逆算して「定時○時以前」を算出する
- **介護の時間制約**:帰宅時間に上限がある場合
- **通院の曜日・時間制約**:特定曜日の午後が使えない等
- **時短勤務の必要性**:フルタイム不可の場合、求人の絞り込みが大幅に変わる
- **通勤時間の上限**:お迎えとの逆算で「勤務地が○○市内なら定時17:30、都内なら定時17:00」のように条件が変わる

**この制約は面談ログの中に埋もれていることが多い。** 子育て中の求職者(特に未就学児がいる場合)には必ず確認すること。

### 検索の基本フロー

1. **職種で絞る**(最初のフィルター)
2. **業種・フリーワードで方向性を絞る**
3. **最低限の条件のみ追加**(土日祝休み、年間休日120日以上など)
4. **一覧で高速スクリーニング**(詳細は見ない)
5. **二次チェック**で条件の詳細・禁忌要素を確認
6. **ABCDマトリックスで仕分け**

### 一覧スクリーニングの目の動き(優先順位順)

1. **必須要件** → そもそもこの求職者が応募できるか(最重要・最初に見る)
2. **求人タイトル・会社名・業種** → 方向性の確認
3. **年収レンジ** → レベル感の一致
4. **仕事内容** → 具体的な業務内容

**原則:「通るかどうか」を先に判断してから「合うかどうか」を判断する。** この順番が検索の生産性を最大化する。

### 年収レンジの判断基準

| 基準 | 例(希望年収500万の場合) |
|---|---|
| **理想の検索レンジ** | 450万〜600万 |
| **許容下限** | 希望年収−100万(400万〜) |
| **例外対応** | 仕事内容が非常にマッチする場合のみ、事前に求職者の了承を得て下げる |

**重要:求職者に無断で年収を大幅に下げた求人を提案しない。例外対応する場合は必ず事前相談。**

### 求職者タイプ別の検索戦略

#### 営業経験者・成長志向(マネジメント志向)

- キーワード:「管理者候補」「マネージャー候補」「マネジメント」
- 業種:本人の経験業種から優先的に探す
- **CAの介入ポイント**:「無形に行きたい」と言っていても、マネジメント志向なら経験業種で探す方が近道。有形商材が嫌な理由を深掘りし、同業種内で解決できるか検討する

#### 営業経験者・成長志向(プレイヤー志向・稼ぎたい)

- キーワードよりも業種で絞る
- 無形商材系の営業を中心に提案
- インセンティブ制度の有無を確認

#### 若手・第二新卒

- フリーワード:「第二新卒」「未経験」
- 必須要件の経験年数が短いものを優先

#### 事務系・安定志向

- 職種:事務系で絞り込み
- 業種:商社・メーカー等の有形商材、老舗企業、設立年数の長い企業
- **固定残業時間による安定度スコアリング**:

| 固定残業時間 | 安定度評価 | 備考 |
|---|---|---|
| 記載なし(0時間) | 実質20時間と想定 | 額面通りに受け取らない |
| 10時間以内 | A(高い安定度) | |
| 20時間以内 | B | |
| 30時間以内 | C | |
| 30時間超 | D(安定志向には不向き) | |

#### 自衛隊・公務員・教師出身者(非ビジネス経歴)

民間企業でのビジネス経験がない求職者は、「何を売りたいか」「どんな業種がいいか」の判断基準を持っていないことが多い。一般的な「無形商材 vs 有形商材」の切り口だけでは不十分で、**本人の素養と志向性から業種を逆引きする**アプローチが必要。

**体力・規律・忍耐力が評価される業種(自衛隊出身で特に有効):**
- **不動産営業**(新築マンション、戸建て、賃貸仲介):稼ぎたい志向が強い場合の第一候補。インセンティブが大きく成果が明確
- **住宅メーカー営業**:新規開拓でありながら企業の安定性も高い。安定×成果達成の複合タイプに最適
- **土地活用・不動産仕入れ**:ゴリゴリの営業だが高年収が狙える。体育会系の素養が直結
- **施工管理**:最も内定が取りやすい「保険」ポジション。免許必須の場合が多い

**注意点:**
- 本人が「営業がしたい」と言っていても、営業の具体的イメージがないことが多い。「体育会系でノルマがきつい」という漠然とした不安を持っている場合は、面談で具体的な仕事イメージを伝えて不安を解消する
- 無形商材(人材・広告・IT)は「第二新卒歓迎」で門戸は広いが、自衛隊出身者の強み(体力・規律・根性)が直接活きにくい場合がある。実際の内定承諾先としては不動産・住宅系の方が決定率が高い傾向
- 業種を最初から絞らず幅広く提案し、2回目の面談で反応を見て絞り込む方が効果的

### 業種提案の優先度設計(S→A→B)

求職者が好きな業界・得意な業界が明確な場合、業種の提案に優先度をつける。

| 優先度 | 定義 | 例 |
|---|---|---|
| **S(最優先)** | 本人が好き・経験がある・志望度が高い業界 | アパレルEC経験者→アパレル会社のEC職 |
| **A** | Sと類似した業界、またはスキルが直結する業界 | アパレル経験者→コスメ・食品等のEC運営 |
| **B** | スキルは活かせるが業界が異なる | アパレル経験者→EC支援会社・メーカーEC部門 |

**重要:本人が業界に強い愛着を持っている場合(「アパレルが好き」等)、SランクをAやBと同列に扱ってはいけない。** 業界愛がある求職者は、年収が多少低くても好きな業界の方が定着率が高く、承諾率も高い。

### 業界NGの「ソフトNG」と「ハードNG」の見分け

求職者が「この業界は避けたい」と言った場合、一律に除外せず濃淡を見極める。

- **ハードNG**:理由が明確で感情的にも拒否が強い(例:不動産は絶対嫌)→ 完全除外
- **ソフトNG**:苦手意識はあるが内容次第では検討可能(例:ITはよくわからないけど…)→ 内容が合う求人を少量混ぜて反応を見る

見分け方:面談で「なぜその業界を避けたいのか」を掘り下げる。具体的なネガティブ体験があればハードNG、漠然とした苦手意識ならソフトNG。ソフトNGの業界から実際に応募が入るケースは多い。

### 実績データを踏まえた紹介戦略(2026年7月・第3回検証で更新)

実運用データの第3回検証(現行評価基準・マイページ送信済み・紹介から3週間以上経過分)で、本人希望ランク別の「送信→エントリー移行率」は以下が確認されている:

- **A評価:約4件に1件がエントリー(27.1%)**
  → 「A=必ず進む」ではない。A評価でも複数件を厚めに紹介し、本人の選択肢を広げる
- **B評価:約6件に1件(16.9%)**
  → A評価と並行してBも提案。Aだけに絞ると紹介数が足りなくなる
- **C評価:約9件に1件(11.5%)**
  → 補助的な提案として少量混ぜる。旧検証(1.6%)の印象より応募されており、Cは「無価値」ではない
- **D評価:ほぼ進まない(4.2%、送信24件中1件)**
  → 保留運用が正しい(Phase 5「Dランクの保留運用」参照)

**紹介量のバランス**:A評価メイン+B評価を厚めに、C評価を視野拡大用に少量、が基本形。

### 「憧れ枠」の戦略的活用(2026年7月検証で新たに確認)

本人希望ランクを固定して比較すると、**通過率ランクが低い(受かりにくい)求人ほど応募されやすい**ことが実データで確認された。

- 本人希望A求人の「応募したい」率:通過率A 10.0% → B 14.0% → C 19.6% → D 30.4%(きれいな単調増加)
- 全16組み合わせ中、最も応募到達率が高いのは「本人希望A × 通過率D」(46.7%)
- ただし応募後の選考進行は通過率ランク通りに厳しい(通過率C〜Dは書類でほぼ止まる)

**活用方針:**

- 求職者は「憧れの求人(希望に合うが通りにくい)」に最も強く反応する。エントリー数と転職活動の温度感を上げたい局面では、憧れ枠を意図的に混ぜる
- ただし憧れ枠だけでは全滅して心が折れる。**憧れ枠(希望A×通過C〜D)+堅実枠(希望A〜B×通過A〜B)のミックス**が基本形(「提案の難易度バランス戦略」の実証版)
- 憧れ枠を出す時は、通過リスクの説明を必ず添える(Phase 5「提案時の注意」)

### ミドル層(35歳以上)の提案戦略

若手層とは根本的にアプローチが異なる。詳細は `references/middle-career.md` を参照。

**核心ルール:**

1. **希望でグリップを握る → 落ちて市場価値を自覚 → 経験業種のより良い環境を提案**
   - 最初から「それは無理です」と言わない。まず希望通りに出してエントリーを重ねる
   - 書類で落ちたタイミングが、市場価値の自覚と提案変更の最大のチャンス
   - 「経験した業界・職種でもっといい環境で頑張りませんか」と現実的な提案に移行

2. **年齢×未経験業種=書類は絶望的**と覚悟する
   - 38歳でB2B未経験・有形商材未経験がメーカー商社を希望 → 書類通過はほぼ不可能
   - ただし「通らないとわかっている求人」も戦略的に出す価値がある(市場理解のため)

3. **退職理由の解像度を上げる**
   - 「業界から出たい」と言っていても、実際は「古い商材の将来性への不安」であり、同業界でもDX系や成長商材なら残った可能性がある
   - 業界嫌いではなく商材嫌いの場合、同業界の成長企業が最適解になりうる

4. **職務経歴書の作り込みが書類通過の最大レバー**
   - ミドル層では推薦状の内容より**職務経歴書の構成と打ち出し方**が書類通過を左右する
   - 「教育業界の営業」ではなく「営業組織マネジメントのプロ」等、ポジショニングの変換が鍵

5. **学歴フィルターへの対応**
   - 大手メーカー・商社は学歴フィルターがかかる可能性。中堅企業・実力主義企業を中心に

### 【一般的CA知見】定量情報 × 定性情報のマッチング

求人マッチングには**定量情報**(年収・勤務地・経験年数など数値化できるもの)と**定性情報**(社風・人柄・価値観などの心理的・感覚的なもの)の両面が必要。

- **定量マッチング**は検索フィルターで対応可能(自動化しやすい)
- **定性マッチング**はCAの経験と判断力で対応する(自動化が難しい)

特に定性マッチングでは以下の観点が重要:
- **組織との相性**:チーム構成、上司のマネジメントスタイル、社風
- **成長環境の適合**:教育体制の有無、OJT vs 体系的研修、自走型か伴走型か
- **働き方の価値観**:ワークライフバランスの捉え方は人によって大きく異なる

### 【一般的CA知見】求職者の「本当のWill」と検索条件のズレ

求職者が言語化する希望条件は表面的であることが多い。実際の応募・内定承諾の決め手は「裏の優先順位」であるケースが多い:

| 表の希望(言語化される) | 裏の優先順位(決め手になりやすい) |
|---|---|
| 年収アップ | 上司との相性、裁量の広さ |
| キャリアアップ | 評価制度の透明性、成長実感 |
| ワークライフバランス | 有給取得のしやすさ、心理的安全性 |
| 業界チェンジ | 今の業務からの脱出(業界は実は何でもいい) |

面談で引き出した情報を元に、表面的な検索条件だけでなく「裏の優先順位」に合致する求人も探すことで、マッチング精度が向上する。

### 実績ベースでの予測精度(第1〜3回検証・最新2026年7月)

本ABCDマトリックスは実運用データで3回検証済み。検証履歴の詳細と数字の読み方は末尾「検証履歴年表」を参照。**最新の確定値(第3回・現行評価基準・マイページ送信済みベース・成熟分)は以下。**

**軸1・本人希望ランク:「応募されるか」を予測する(3回連続で順序再現)**

送信→エントリー移行率:

| ランク | 移行率 | サンプル数 |
|---|---|---|
| A | 27.1% | 643件 |
| B | 16.9% | 852件 |
| C | 11.5% | 366件 |
| D | 4.2% | 24件(保留運用によりサンプル僅少) |

**軸2・通過率ランク:「応募後に選考が進むか」を予測する(2026年7月に初めて実証)**

エントリー→書類通過以降への進行率:

| ランク | 進行率 | サンプル数 |
|---|---|---|
| A | 30.9% | 123件 |
| B | 21.9% | 260件 |
| C | 10.3% | 97件 |
| D | (参考)14.3% | 21件(分子3件のみ・順序判定保留) |

**2軸の役割分担の実証(最重要の知見)**

- 本人希望ランクは**応募意欲を予測するが、選考進行は予測しない**(本人希望別の進行率はA〜Dほぼ横並び)
- 通過率ランクは**選考進行を予測するが、応募意欲は予測しない**(むしろ逆相関:受かりにくい求人ほど応募したがる。Phase 4「憧れ枠」参照)
- 2軸は独立に機能している。**並べ替え・優先度判断は総合ランク単独ではなく、本人希望×通過率の2軸マトリクスで行う**(下記「総合評価と2軸マトリクス運用」参照)
- 同一求職者内の比較でも本人希望A>B>Cの差は再現済み(同じ求職者の中でA vs C:+17ポイント)。「行動量の多い求職者にAを多く出していただけ」という交絡は検証で否定された

**数字の読み方**

- 進行率は右打ち切り(直近応募分の選考結果待ち)を含む暫定値で、今後上振れしうる。順序のみ信頼する
- 過去の検証値(第1回19.2%等)との絶対値比較は禁止。評価基準・集計方式が異なる(末尾年表参照)

### Aランクでも進まない理由

求人紹介Aランクは「マッチ度が高い求人」を意味しますが、5件紹介しても1件しかエントリーに進まないのが実態です。これには2種類の理由があります。

**求職者側の事情(AIの精度とは無関係)**
- 紹介後に「やはり今は転職をやめる」と言ってくる
- 他社で既に内定が出て決定済み
- 連絡が途絶える、面談に来なくなる
- 紹介したばかりでまだ本人が判断中

**AI側の事情(これは精度の問題)**
- 本人の希望条件は満たしているが、本人の好み・温度感を読み違えている
- 過去の経歴から推測した希望と、本人の今の気持ちにズレがある

### 評価の二軸

**軸1:本人希望適合度**

| ランク | 定義 |
|---|---|
| A | 志向性も条件もほぼ一致。本人が間違いなく応募する求人 |
| B | 方向性は合っているが、条件が何か足りない |
| C | 条件は適しているが、方向性が合っていない |
| D | 両方適していない |

**軸2:選考通過率**

**最初に必ず確認:学歴・資格・経験年数の必須要件チェック**
求人に「大卒以上」「○○資格必須」「経験○年以上」と記載がある場合、求職者がこれを満たしていなければ通過率は自動的にD判定。他の条件がどれだけ合っていても必須要件未充足でA・B判定をつけることは絶対にあってはならない。推薦状でカバーできる可能性がある場合のみC判定を検討し、その旨を明記する。

| ランク | 判断材料 |
|---|---|
| A | 必須要件を全て満たした上で+過去の通過実績あり+企業の採用温度感が高い |
| B | 必須要件をクリア+一部不確定要素あり |
| C | 必須要件は満たすが通過率に不安要素あり、または必須要件未充足だが推薦状でカバー可能性あり |
| D | 必須要件の充足が不十分(学歴・資格・経験年数のいずれかが未充足) |

### 通過率の判断材料(優先順)

1. **求人票の必須要件と求職者スペックの一致度**(年齢・経験年数・スキル)— 最重要
2. **自社求人であれば企業への応募状況確認**(温度感・競合候補者の有無)
3. **過去の通過率データ**(蓄積されていれば)
4. **タイミングの変数**(同じスペック・同じ求人でも応募倍率で結果が変わる)
5. **経験の質(自社運営 vs 業務委託)**:同じ「EC運営5年」でも、自社でECを回している経験と、業務委託で一部業務を切り出して担当している経験では書類評価がまったく異なる。業務委託・BPO・派遣での経験は、経験年数ほど通過率が高くならないことがある

### Dランクの保留運用(正式仕様・2026年7月明文化)

**D評価が付いた求人は、原則として紹介保留とする。** これは実データで裏付けられた正式な運用である:

- 実際の送信率:A〜B評価は73〜79%が求職者に送信されるのに対し、D評価は21〜28%のみ。保留判断はD〜Cにグラデーション状に効いている
- 保留を押し切って送信されたD(CAが個別判断で救済した24件)ですら、エントリーは1件(4.2%)のみ
- 過去の混合期間の集計では「送信されたDは意外に応募される」ように見えた時期があったが、期間分割検証の結果、旧評価基準時代のわずか6件が作った見かけ上の現象と確定した。現行基準では再現しない

**帰結:**

- D評価の実績データは今後も蓄積されない(保留するため)。これは欠陥ではなく運用の帰結。**Dランクの移行率の数字は精度評価に使わない**
- 例外的にDを送る判断(本人が会社名指しで希望した等)は従来通りCAの裁量。ただし「Dでも応募されるかもしれない」という期待は、現行基準の実データで否定されている

### 提案の難易度バランス戦略

自信がない求職者(前回の転職失敗経験がある、打たれ弱いタイプ等)に対しては、**難しい求人と通る求人を半々で提案する**のが最適。

- **通りやすい求人だけ出す**:いつまでも市場価値や自分のレベルを正しく理解できず、条件を下げない
- **難しい求人だけ出す**:全部落ちて心が折れる
- **半々で出す**:不合格で現実を理解しつつ、通過で自信を維持。最も学習効果が高い

### 総合評価と2軸マトリクス運用(2026年7月改訂)

本人希望ABCDと選考通過率ABCDを掛け合わせて総合ABCDを算出する。ただし**実データ検証の結果、紹介の優先度判断は総合ランク単独ではなく、2軸マトリクスで行うことを標準とする。**

理由:総合ランクは2軸の合成であるため、予測の切れ味が両軸単独より鈍ることが確認された(応募後の進行率で総合A<総合Bの逆転が安定して発生)。「応募されるか」は本人希望、「受かるか」は通過率が、それぞれ最も正確に予測する。

| 希望 × 通過 | 位置づけ | 提案方針 |
|---|---|---|
| A × A〜B | **主力(堅実枠)** | 最優先で提案。エントリーも選考進行も見込める |
| A × C〜D | **憧れ枠** | 最も応募されやすいが選考は厳しい。通過リスクの説明を添えて意図的に混ぜる(Phase 4「憧れ枠」参照) |
| B × A〜B | 主力の脇 | 「受かりやすく方向性も悪くない」として厚めに提案 |
| C × A | 気づき枠 | CAの提案力で「実は合うかもしれない」と視野を広げる補助 |
| D × 任意 | 原則保留 | 「Dランクの保留運用」参照 |

総合ランクは一覧の初期並び順などの簡易表示には引き続き使ってよいが、紹介判断の根拠には2軸を使う。

### 【一般的CA知見】第三軸:定着可能性

ABCDの二軸に加え、トップCAは**入社後の定着可能性**も考慮する。内定承諾に至っても早期離職になれば、求職者・企業・CA全員にとってマイナス。

定着可能性の判断要素:
- **組織カルチャーとの適合度**:体育会系 vs フラット、トップダウン vs ボトムアップ
- **前職の離職理由と転職先の環境の整合性**:同じ理由で辞めるリスクがないか
- **キャリアパスの現実性**:入社後に期待する成長が実現できる環境か
- **生活面の持続性**:通勤距離、年収変動の許容範囲、家庭の状況

### 提案時の注意

- **求職者にはABCDランクを見せない**。信頼関係を守るため
- Aランクの求人は「おすすめ求人」「ピックアップ求人」として提案
- ABCDの読みが外れた場合に信頼を損なわないよう、内部の判断基準は開示しない
- 提案時に、求職者がつまずきそうなポイントを先回りして説明を添える(エントリー率向上)

## Phase 6: 提案後の対応とクロージング

### 求職者が応募を躊躇する場合の対応

#### パターン1:自己過小評価(最も多い)

求職者が必須要件の文言に過剰反応して「自分には無理だ」と思い込む。

- 例:「PCスキル一定以上」→「パソコンは入力程度しかできない」
- 例:「○○の知識がある方」→ 過剰な不安
- **対応**:「このレベルで大丈夫ですよ」と具体的に説明→エントリーに繋がるケースが多い

#### パターン2:求人票の細部の見落とし

仕事内容は気に入っているが、固定残業時間や勤務地の詳細など細部の不一致で引っかかる。

- **対応**:先回りして「ここはこういう条件ですが問題ないですか?」と確認を添える

#### 【一般的CA知見】パターン3:漠然とした不安

「なんとなく違う気がする」「ピンとこない」という曖昧な拒否。これは求職者自身の判断基準が不明確なサイン。

- **対応**:「具体的にどこが引っかかりますか?」で掘り下げる。出てこない場合はWill-Can-Mustに立ち返り、優先順位を再確認する

### 内定後のクロージング

#### 承諾を迷っている場合のアプローチ

**感情論ではなく、データと確率で現実を提示する。**

1. 現在の選考状況を数字で整理する(何社応募→何社通過→通過率X%)
2. この内定を手放した場合のシミュレーションを提示:
   - 同等の求人を見つけるために必要な応募数
   - その求人が見つかる確率
   - 見つかったとしてもその求人が本当に希望に合うかの不確実性
   - 転職活動期間の延長リスク
3. 最後に逃げ道を用意する:「長期的にやっていくならそれでもいいと思いますよ」

**重要:押し売りにならないよう、求職者自身に合理的な判断をさせる。**

#### 競争感の演出

- 企業に応募状況を確認し、他候補者が選考中であることを伝える
- 枠が埋まりそうな状況があれば事実として共有する
- 年収交渉は企業側の温度感が相当高くない限り基本的に行わない

### 【一般的CA知見】面接対策とフォローアップ

マッチング精度を高めるのは求人選定だけではない。**面接対策**もマッチングの一部。

- **企業が求める人物像**を把握し、求職者の経験の中でそこにフィットするエピソードを一緒に整理する
- **推薦状(推薦コメント)**に面談で引き出した求職者の魅力を的確に反映する
- **面接後のフォロー**で求職者・企業双方の温度感を確認し、ギャップがあれば早期にケアする

## Phase 7: 選考プロセスのリスク管理

### 求人票の「裏の基準」への対応

- 必須要件を満たしていても落ちるケースは頻繁にある
- 求人票に書かれていない暗黙の基準が存在する:
  - 年齢の上限感(「第二新卒歓迎」でも実質25歳までの場合あり)
  - 前職の企業規模感(大手出身を暗に求める場合あり)
  - 転職回数の許容範囲
  - 直近の在籍期間(短期離職への警戒)
- **必須要件のみで選考の角度を読み取るのは危険**
- 事務局による機械的な足切りで、スキル的には十分でも弾かれるケースがある

### タイミングの変数

同じスペック・同じ求人でも、その時点での応募倍率や競合候補者の状況で結果が変わる。これはCAがコントロールしにくい変数だが、「今この求人は応募が少ないから通りやすい」という情報を持っていれば、提案の優先順位を変える武器になる。

### 求職者は求人票を正しく読めていない

求職者の多くは求人票の読解力が十分ではない。必須要件への過剰反応、条件の見落としが頻繁に発生する。提案時に以下の対応を取る:

- 求職者がつまずきそうなポイントを先回りして説明を添える
- 「この求人のここが合う理由」を言語化して提示する
- 必須要件の文言を噛み砕いて「あなたの経験で言うとこういうこと」と翻訳する

### 【一般的CA知見】お見送りデータの活用

お見送り結果は最も学習価値の高いデータ。以下の分析を蓄積する:

- **パターン検出**:「この企業はいつも○○の理由で落とす」「この職種は△△が本当の基準」
- **フィードバック精度**:企業からの見送り理由は建前であることも多い。複数回の結果を照合して真の基準を推測する
- **市場感の更新**:通過率の変動から市場の温度感を掴む

**現状の蓄積状況での留意点(2026年5月時点)**:
- 書類選考以降のサンプルは現時点で限定的(通過率ランクA評価で書類段階の結果確定は2〜3件のみ)。お見送りパターンの統計的分析は3〜6ヶ月後の再集計まで保留
- AI再評価ケース(過去に分析済みの求人を再分析しているもの)が稀に発生する。「前回分析済み・判定変更なし」と AI コメントに含まれるケースは予測精度の検証データとしては無効

## データ蓄積と精度向上

### 蓄積すべきデータ(1ケースあたり)

**構造化ファクト(スキル学習に使用):**
- 求職者の基本スペック(年齢、経験職種、経験年数、スキル、転職回数)
- タイプ診断結果(主タイプ + 副タイプ)
- Will-Can-Must分析結果
- 希望条件リスト(表の希望 + 裏の優先順位)
- 提案した全求人リスト
- 各求人のABCD評価(希望適合度・選考通過率・総合)
- 応募した求人と選考結果(書類通過・面接通過・内定・承諾or辞退)
- 最終的な転職先と決め手

**参照用データ(フル保存):**
- 面談ログ全文テキスト
- 経歴書・職務経歴書(PDF)

### 蓄積の原則

- 人の感想や解釈ではなく、**事実と結果**を中心に記録する
- 企業の落選理由は正確に拾えないため深追いしない
- 「このスペック × この求人 = この結果」をシンプルに蓄積し、件数で確率論的に精度を上げる
- 100件超で業界・職種別のパターンが見え始め、500件超で統計的に信頼できる傾向が出る

### 精度向上のサイクル

1. データ蓄積 → 2. パターン抽出(どんな人×どんな求人が成功するか)→ 3. スキルルールの更新 → 4. 検索戦略の改善 → 1に戻る

このサイクルを回すことで、経験則が構造化された知識に昇華され、属人的な「勘」がチーム全体で共有可能な「スキル」に変わる。

### 検証時の注意点(2026年5月時点で確認された運用知見)

実運用データで AI 評価の予測精度を検証する際、以下を踏まえる必要がある:

**1. リードタイムの存在**
- 求人紹介から本人エントリーまで数日〜数週間のタイムラグがある
- 検証は「紹介から十分時間が経った成熟データ」で行う必要がある
- 直近1ヶ月以内に紹介した求人を母数に含めると、「まだ進む時間が経っていない」だけのケースが空振りとして数えられ、予測精度が過小評価される

**2. AI再評価ケースの除外**
- 過去に AI 評価済の求人を再分析しているケースがある(「前回分析済み・判定変更なし」と AI コメントに含まれる)
- これは「結果が出た後の事後分析」になり、予測検証データとしては無効
- 集計時には除外する

**3. 求職者離脱の影響**
- 紹介後に「やはり転職しない」と本人が判断するケースは一定割合発生する
- これは AI の精度問題ではなく求職者要因なので、全ランクに均等にノイズとして混ざる前提で集計する
- 結果として、ランク間の比較精度には影響しない

**4. CA保留運用の除去(2026年7月追加・必須)**
- 全紹介を母数にすると、D・C評価の数字がCAの保留判断(送らない運用)で歪む
- 検証は必ず「マイページ送信済み(lastExportedAt非null)」を母数にする。第1・2回検証はこれが混ざっており、「AI精度×CA運用」の合成値だった

**5. 判定基準の「章」をまたぐ比較の禁止(2026年7月追加・必須)**
- AI評価の判定基準(評価プロンプト・本スキルの内容)は改訂のたびに「章」が変わる
- 異なる章の数字の絶対値を比較しない。章をまたいで比較してよいのは順序(A>B>C>D)のみ

|---|---|---|---|

**判定用版に入れてはいけないもの(意図的な除外・補完禁止):**

- **実績値・的中率**:判定AIが自らの成績表を読むと、それに合わせてランクの付け方を自己調整する。すると次回検証で「精度が上がった」のか「基準が動いただけ」かが区別できなくなる
- **CA向けの紹介戦略**(憧れ枠・提案ポートフォリオ等):判定AIが読むと「通りにくいが応募されそうだから本人希望を上げる」といった軸をまたいだ忖度が発生し、2軸の独立性が壊れる。2軸が独立していることが本スキルの中核的価値
- **検証履歴年表**:判定に無関係

**更新手順:**

1. フル版を更新（本ファイル）
2. フル版から上記3種を除いた判定用版を作り、portal に反映（AIプロンプト変更に該当するため staging 検証必須）
3. 反映日を上記「検証履歴年表」に**新章の開始**として追記する
