## 1. モックアップ HTML 構造サマリー

### 全体レイアウト
- `im-wrap`: max-width 1600px, 全画面高さ, flex column
- `im-hdr`: ヘッダーバー（タイトル + パンくず + アクションボタン群）
- `im-grid`: **CSS Grid 2カラム均等分割** (`grid-template-columns: 1fr 1fr`)
- カスタム CSS 変数による独自デザインシステム（Tailwind 不使用）

### ヘッダー
- 左: 「面談履歴入力」タイトル + パンくず「求職者詳細 / 福森 志帆 / 面談 #1」
- 右: 4ボタン（← 一覧に戻る / キャンセル / PDF表示 / 保存）

### 左カラム
3セクション、各セクションヘッダーは `border-left: 3px solid blue` + 背景色付き:

1. **面談基本情報** — 6列グリッド (`grid-template-columns: repeat(6, 1fr)`)
   - 面談日, 時刻(開始+終了), 時間/手法(duration + select)
   - 求職者ID(RO), 氏名(RO), フリガナ(RO)
   - 生年月日(RO), 電話(RO), メール(RO)
   - 年齢/性別(RO), 住所(RO, span 4列)
   - 担当CA(RO), 社員名(RO), ランク(バッジ: B+)
   - 回数/状態(RO + chip), 結果(select), 最新(chip)

2. **転職活動状況** — 行ベースレイアウト (`.r` = flex row)
   - 他AG状況: select + input(メモ)
   - 転職時期: select + input(メモ)
   - 活動期間: select + input(メモ)
   - 他社応募: select + input(メモ) + 数値入力「N社」
   - 最終学歴: select + input(学校名) + 卒業年月 + select(卒業/中退)

3. **職務経歴** — `portal-frame` (スクロール可能エリア) + カード形式
   - 各カード: 削除ボタン, N社目, 企業名, 在籍年月
   - 会社概要, 職種, 業務内容(textarea), 退社理由(3段), 詳細(textarea)
   - 「＋ 職歴を追加」ボタン

### 右カラム
5タブ切り替え + メモセクション:

1. **初期条件タブ**
   - 登録時条件: 5列グリッド (業種1/2, 職種1/2, エリア+雇用形態+年収)
   - メモセクション: カード形式CRUD (タイトルinput + タイプselect + 日付 + 時刻 + textarea)
   - 「＋ 新規メモ登録」ボタン

2. **希望条件タブ**
   - サブタブ(職種/業種/エリア) — 各 input + textarea
   - 年収・勤務条件: 現年収/希望下限/希望年収/休日/残業/転勤 (各 select + input)
   - スキル: 免許/語学/日本語/Typing/Excel/Word/PPT (各 select + input)
   - 働き方: 4列チェックボックスグリッド (12項目)

3. **ランク評価タブ**
   - ランクボックス: 大きなランク文字 + 合計点
   - 評価テーブル: 3カテゴリ × 5項目 = 15行
     - 人物評価: 転職意欲/コミュニケーション/ビジネスマナー/地頭/人間性
     - 経歴評価: 経験職種/社会人経験/転職回数/実績・スキル/語学・資格
     - 条件評価: 希望職種/希望年収/休日・シフト/エリア/柔軟性
   - 各行: 1-5ボタン + 備考input
   - 小計行 (subtotal) + 合計行 (total, 黄色背景)

4. **アクションタブ**
   - 応募書類状況: 書類状況select + メモ, サポートselect + メモ
   - 連絡方法: 連絡手段select + メモ
   - 求人送付/送付期限: 送付予定select + メモ
   - 次回面談予定: 日時select + date + time + メモ
   - ネクストアクション: textarea + 「✨ AI整理」ボタン

5. **添付タブ**
   - アップロードエリア: ドラッグ&ドロップ + ファイル選択
   - 添付ファイル一覧: カード形式 (削除, アイコン, ファイル名input, タイプselect, 日付, サイズ, ステータスchip)
   - 「✨ ログを解析して各カラムへ自動入力」ボタン
   - 各カードにメモtextarea

## 5. 実装計画

### 方針: 新ファイル `InterviewFormV2.tsx` を作成し、段階的に置き換え

現在の `InterviewForm.tsx` (1012行) を直接書き換えるのはリスクが高いため、
`InterviewFormV2.tsx` を新規作成し、`InterviewHistoryTab.tsx` の import 先を切り替える。
旧ファイルはバックアップとして一時的に残す。

### 実装フェーズ

**Phase A**: ヘッダーバー + 左右カラムのシェル構造
- ヘッダー（タイトル + パンくず + アクションボタン4つ）
- CSS Grid 2カラム均等分割
- セクションヘッダーのスタイル（border-left + 背景）
- 既存ロジック (state, fetch, autosave, save) をそのまま移植

**Phase B**: 左カラム — 面談基本情報
- 6列グリッドレイアウト
- 求職者基本情報 (RO) の表示（candidate props から取得）
- ランクバッジ、回数/状態chip
- 面談日/時刻/手法のフィールド配置をモックアップ準拠に

**Phase C**: 左カラム — 転職活動状況 + 職務経歴
- 転職活動状況: モックアップ準拠の行レイアウト (select + input)
- 職務経歴: カード形式（※複数社対応はDBスキーマ制約あり、現スキーマでは1社のみ保存可能な場合はUI上1社で実装）

**Phase D**: 右カラム — タブ切り替え全5タブ
- 初期条件: 登録時条件グリッド + メモCRUD
- 希望条件: サブタブ + 年収・勤務条件 + スキル + チェックボックスグリッド
- ランク評価: テーブル形式 + ランクボックス
- アクション: モックアップ準拠 + ネクストアクション + AI整理ボタン
- 添付: 既存ロジック温存 + UI をモックアップ準拠に

**Phase E**: InterviewHistoryTab の import 切り替え + 動作確認
- `InterviewForm` → `InterviewFormV2` に切り替え
- props の互換性確認
- ビルド + 型チェック + 動作確認

### 注意: candidate 情報の受け渡し

モックアップでは求職者基本情報（氏名/フリガナ/電話/メール/住所等）が左カラムに表示されるが、
現状の `InterviewForm` は `candidateId` しか受け取っていない。
`InterviewHistoryTab` → `InterviewFormV2` に candidate オブジェクトを渡す props 追加が必要。

### 注意: 職務経歴の複数社対応

モックアップでは複数社のカード形式だが、Prisma スキーマの `InterviewDetail` に
職歴関連フィールドが1社分しかない可能性がある（companyName, tenure 等が単一フィールド）。
スキーマ変更は禁止のため、UI上は1社表示 + 将来拡張のコメントとする。

### 注意: メモ CRUD

現状のメモは API 経由で取得・表示のみ。モックアップではインラインCRUDが必要。
既存 API: `POST /api/interviews/[id]/memos`, `DELETE /api/interviews/[id]/memos/[memoId]`
これらの API を活用してCRUD UIを実装する。
